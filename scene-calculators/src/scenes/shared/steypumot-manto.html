3. Calculations

3.1 Panel selection (Grunneiningar only)

Use only rows from the Grunneiningar group as panels. For panel area, use the m2 column directly (no guessing).

For a given formworkHeightM:

  • If formworkHeightM = 3, prefer panels whose description contains x300.
  • If formworkHeightM = 1.2, prefer panels with x120.

Algorithm (simple greedy by panel area):

// 1. Filter panels that match height
const panels = allItems.filter(
  i => i.group === "Grunneiningar"
    && i.areaM2 > 0
    && (formworkHeightM === 3
         ? i.name.includes("x300")
         : formworkHeightM === 1.2
           ? i.name.includes("x120")
           : true // fallback: allow all if height does not match standard
       )
);

// 2. Sort by area descending for greedy fit
panels.sort((a, b) => b.areaM2 - a.areaM2);

// 3. Greedy selection
let remainingArea = totalAreaM2;
const panelQuantities = new Map<string, number>(); // key = erlVoruNr

for (let i = 0; i < panels.length; i++) {
  const p = panels[i];
  const area = p.areaM2;
  if (area <= 0) continue;

  if (i < panels.length - 1) {
    // For all except the smallest panel: take as many whole panels as possible
    const count = Math.floor(remainingArea / area);
    if (count > 0) {
      panelQuantities.set(p.erlVoruNr!, count);
      remainingArea -= count * area;
    }
  } else {
    // For the last (smallest) panel: if any area remains, cover it with 1 panel
    if (remainingArea > 0) {
      const prev = panelQuantities.get(p.erlVoruNr!) || 0;
      panelQuantities.set(p.erlVoruNr!, prev + 1);
      remainingArea = 0;
    }
  }
}

This ensures the sum of panel areas (using the m2 field) is greater than or equal to totalAreaM2 and uses only the panel types you have defined.

3.2 Accessories & connectors quantities

For Samtengihlutir and Aukahlutir, quantity should be derived from total area and/or total number of panels, using configurable factors (set by you). The calculator should expose these factors in a configuration object, not hard-coded magic numbers. 

type MantoFactors = {
  // examples, actual values decided by you:
  connectorsPerPanel: number;  // e.g. 2 Manto Flekaklemmur per panel
  tiesPerM2: number;           // e.g. 0.5 ties per m²
  bracesPer10m: number;        // e.g. 1 brace per 10 m wall
};

// After computing panelQuantities:
const totalPanels = Array.from(panelQuantities.values())
  .reduce((sum, c) => sum + c, 0);

const totalArea = totalAreaM2;

// Example quantity rules (to be refined by you):
// - Flekaklemmur: factors.connectorsPerPanel * totalPanels
// - Mótarær DW15: factors.tiesPerM2 * totalArea
// - Ítalskar mótaskástífur: factors.bracesPer10m * (totalArea / formworkHeightM / 10)

The agent must not invent the factors themselves; they are configuration inputs (or read from the Excel if you later add them there).

3.3 Pricing

For any item with a quantity q and a daily price with VAT dailyPriceWithVat: 

lineDailyWithVat   = q * dailyPriceWithVat;
lineDailyExVat     = lineDailyWithVat / (1 + vatRate);
lineTotalWithVat   = lineDailyWithVat * rentalDays;
lineTotalExVat     = lineTotalWithVat / (1 + vatRate);

Overall totals: 

totalDailyWithVat = sum(lineDailyWithVat for all items);
totalDailyExVat   = sum(lineDailyExVat for all items);
totalWithVat      = sum(lineTotalWithVat for all items);
totalExVat        = sum(lineTotalExVat for all items);

4. React component

Implement a React component MantoFormworkCalculator.tsx:

  • React 18, TypeScript, functional component with hooks.
  • Inputs:
    • totalAreaM2
    • formworkHeightM (defaults to 3)
    • rentalDays
    • vatRate
    • MantoFactors (either hard-coded or editable UI).
  • On change, call calculateMantoOffer and render:
    • Table of all items with quantity and prices.
    • Summary with total area and totals ex./inc. VAT.
